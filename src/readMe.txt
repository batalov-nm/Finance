Тест-кейсы:

1)
Название
    проверка на ввод пустого поля(Вкладка Юр. Лица)
Описание:
    Во вкладке Юр лица при попытки создать юр лица с пустыми полями, сайт должен выдавать ошибку
Воспроизведение:
    0) Зайти на сайт и произвести авторизацию
    1) Зайти во вкладку юр лица
    2) Нажать кнопку "Добавить юр лицо"
    3) Оставить поля пустыми
    4) Нажать кнопку - Добавить
Желаемый результат:
    Сайт выдаст ошибку
Реальный результат:
    Сайт выдал ошибку

2)
Название -
    Работоспособность кнопки Добавить в разделе счета
Описание -
    Во вкладке Счета есть возможность добавить новый счет. При корректно введенных данных новый
    счет должен появлятся в списке счетов
Воспроизведение -
    0) Авторизация на сайте
    1) Зайти на раздел Счета
    2) Нажать кнопку Добавить счет
    3) Заполнить пустые поля
    4) Нажать кнопку добавить
    5) Счет должен появиться в списке счетов
Ожидаемный результат -
    Счет появлися в списке счетов
Реальный результат -
    Счет появлися в списке счетов

3)
Название -
    Возможность создания нового пользователя
Описание -
    На сайте присутствует возможность создать нового пользователя. При введении Почты правильного формата и пароля,
    подходящим по критериям, создается новый польщователь
Воспроизведение -
    0) Авторизация на сайте
    1) Зайти на вкладку Пользователи
    2) Нажать на кнопку Добавить нового пользователя
    3) Ввести корректные данные правильного формата в пустые поля
    4) Выбрать роль Пользователя
    5) Нажать кнопку Добавить
    6) Новый пользователь создан
Ожидаемный результат -
    Новый пользователь создан
Реальнй результат -
    Новый пользователь создан

4)
Название-
    проверка сортировки при помощи фильторв во вкладке Платежи
Описание -
    Во вкладке платежи существует список с различными платежаии, сверху этого списка
находятся фильты по которым можно упорядочить список. Данный список должен производить сортировку списка
Воспроизведение -
    0) Зайти на сайт и провести авторизацию
    1) Зайти на фкладку Платежи
    2) Нажать на любой фильтр и выбрать любое значение из выпадающего окна
    3) Проверить, изменилась ли сортировка списка
    4) Проверить правильность сортировки списка
Желаемый результат -
    Сортировка списка произведется по параметру фильтра
Реальный результат -
    Сортировка не производиться ни по одному фильтру

5)
Название -
    Невзможность добавления платежа с отрицательным значением
Описание -
    При создании нового платежа, сумма данного платежа не должна принимать отрицательные значения
Воспроизведение -
    0) Авторизация на сайте
    1) Зайти в раздел Платежи
    2) Создать новый платеж с отрицательным значением суммы
Желаемый результат -
    Платеж с отрицательным значением не создаться
Реальный результат
    Платеж с отрицательным значением создать можно
